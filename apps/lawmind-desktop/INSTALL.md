# LawMind 安装说明（终端用户）

面向「下载压缩包 / 安装包即用」的律师工作台。**无需单独安装 Node.js**：安装包内已附带运行本地服务所需的 Node 运行时。

## 我该下载哪一个？

在发布页（或供应商提供的网盘）按自己的电脑选择 **对应系统、对应架构** 的文件。文件名形如：

`LawMind-<版本>-<系统>-<架构>.<扩展名>`

| 系统 | 常见文件 | 使用方式 |
|------|----------|----------|
| **Windows（64 位）** | `.exe`（安装向导）、`…-portable.exe`（绿色版）、`.zip`（解压文件夹） | 双击安装或解压后运行其中的 `LawMind.exe`。可选：将 zip 解压到固定目录后建桌面快捷方式。 |
| **macOS（Apple 芯片 / Intel）** | `.dmg` 或 `.zip` | **zip**：解压后将 `LawMind.app` 拖进「应用程序」。首次若被拦截：**右键 → 打开**。未签名测试版可能需在「系统设置 → 隐私与安全性」中允许。 |
| **Linux（x86_64）** | `.AppImage` 或 `.tar.gz` | **AppImage**：`chmod +x LawMind-*.AppImage` 后双击或终端运行。**tar.gz**：解压到目录后运行其中的可执行文件（名称以包内为准）。 |

> **架构不匹配**会导致无法运行，请与「关于本机 / 系统信息」中的处理器架构一致。

## 更新 LawMind

- **应用内更新（打包版）**：联网时应用会在后台检查新版本；也可在菜单 **帮助 → 检查更新** 或 **设置** 底部「应用更新」中手动检查。依赖与正式安装包相同的 **GitHub Release**（见开发者文档中的 `latest*.yml` 说明）。
- **手动下载**：菜单 **帮助 → 下载安装包** 或浏览器打开「下载落地页」，按系统下载对应压缩包或安装程序后覆盖安装即可。

## 首次打开以后要做什么？

1. 按向导填写 **大模型 API Key**（及可选 Base URL、模型名、工作区路径）。配置写入本机用户目录下的 LawMind 数据区。
2. 在 **设置** 中可 **创建多个智能体**（不同分工）；需要多步衔接时可在 **协作与多智能体流程** 中配置后台工作流。
3. **对外文书、材料交付**：请在顶部 **审核** 中完成律师把关与门禁后再渲染或对外发出。

## 常见问题

- **防火墙 / 安全软件**：本地服务仅监听 **127.0.0.1**；若被拦截，请按单位 IT 策略允许。
- **模型报错或 502**：检查 API Key、网络及模型服务商状态；应用内会尽量展示服务端返回的具体原因。

## 隐私与责任

数据处理说明见 [LAWMIND-DATA-PROCESSING](https://docs.lawmind.ai/LAWMIND-DATA-PROCESSING)。输出为辅助草稿，**不构成法律意见**。

## 企业批量部署（IT）

- **macOS**：广泛分发建议 **代码签名 + 公证**；内嵌 Node 需符合安全策略。
- **Windows**：建议 **Authenticode 签名** 以减少 SmartScreen 提示。
- 供应链：仓库侧可生成 SBOM（根目录 `pnpm lawmind:sbom`、`pnpm lawmind:sbom:cyclonedx`）。

开发与开发者打包步骤见 [README.md](./README.md)。
