# LawMind 联网检索路径说明（开发选型）

本文档说明 LawMind 中「公网检索 / 联网」相关的实现路径，供后续产品与工程选型参考。

## 重要区分

**聊天 API（OpenAI-compatible，如 DashScope compatible-mode）本身不等于自带公网网页搜索。**  
用户勾选「允许联网检索」时期望的是「应用内一键搜网页」；这需要在应用侧单独接入搜索能力或厂商提供的联网插件，而不是仅配置聊天模型。

## 路径对照

| 路径                        | 说明                                                                                                                                                                                         |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **A. 当前实现**             | 勾选「允许联网检索」时注册 `web_search` 工具，使用 **Brave Search API**。密钥：`LAWMIND_WEB_SEARCH_API_KEY` 或 `BRAVE_API_KEY`（与 OpenClaw Brave 配置兼容）。与用户配置的「聊天模型」解耦。 |
| **B. 厂商原生联网**         | 若 DashScope 或其它网关提供**官方、稳定的「带联网」模型或插件 API**，可在 LawMind 侧对接，替代或与 Brave 并存（需评估计费、合规、地域）。                                                    |
| **C. 与 OpenClaw 主栈对齐** | 复用仓库内 `src/agents/tools/web-search-core.ts` 及主配置里 `tools.web` 的解析逻辑。工作量大，依赖主配置体系，适合与完整 OpenClaw 网关/配置打通时使用。                                      |

## 桌面应用中的行为

Electron 桌面应用在主界面提供「允许联网检索」开关；未勾选时不会注册联网搜索工具。详见 [LawMind 使用手册](/LAWMIND-USER-MANUAL) 桌面应用章节。
