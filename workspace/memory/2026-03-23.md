## 今日工作摘要

实现了 LawMind 助手间智能协作系统（Inter-Assistant Intelligence System），使多个法律助手能根据律师指令相互沟通、委派任务、咨询和审查，达到类人级别的协作交互。

---

## 完成项

### 1. 协作类型系统 (`src/lawmind/agent/collaboration/types.ts`)

- 定义 `CollaborationMessage`（5种消息类型：delegate/consult/notify/review_request/result）
- `DelegationRecord`（生命周期：pending → running → completed/failed/timeout）
- `ReviewType`/`ReviewFeedback`（审查类型与结构化反馈）
- `CollaborationEvent`（11种审计事件）
- `CollaborationPolicy`（限流、深度控制、通信白名单）

### 2. 消息总线 (`src/lawmind/agent/collaboration/message-bus.ts`)

- 基于 OpenClaw fire-wait-read 模式的助手间消息路由
- 同步模式（`sendAndWait`）：咨询/审查场景，调用方阻塞等待回复
- 异步模式（`fireAndForget`）：委派/通知场景，返回 completion Promise
- 不可信内容包装（防止 prompt injection）
- 动态解析目标助手配置（profile → AgentConfig）

### 3. 委派注册表 (`src/lawmind/agent/collaboration/delegation-registry.ts`)

- 参照 OpenClaw subagent-registry.ts 设计
- 内存 Map + JSON 磁盘持久化（`workspace/delegations/`）
- 生命周期管理：create/running/completed/failed/timeout/cancelled
- 策略执行：最大活跃数、嵌套深度限制、通信白名单校验
- 进程重启后自动恢复（`restoreDelegationsFromDisk`）

### 4. 六个协作工具 (`src/lawmind/agent/tools/collaboration-tools.ts`)

- `delegate_task`：异步委派子任务给另一个助手
- `consult_assistant`：同步咨询另一个助手
- `notify_assistant`：向助手发送通知（fire-and-forget）
- `request_review`：请求审查（同步，支持4种审查类型）
- `list_delegations`：查看委派任务状态
- `get_delegation_result`：获取委派完整结果
- 所有工具支持按 ID 或显示名称查找目标助手

### 5. 运行时集成

- `AgentContext` 扩展 `collaborationEnabled` 和 `collaborationDepth` 字段
- `AgentConfig` 扩展 `enableCollaboration` 选项
- `ToolDefinition.category` 新增 `"collaboration"` 类别
- `createLegalToolRegistry()` 支持 `enableCollaboration` 参数动态注册协作工具
- `buildSystemPrompt()` 新增协作规范段（工具说明 + 可协作助手列表 + 行为规范）
- `runTurn()` 自动收集可协作的 peer 助手信息注入 system prompt

### 6. 协作 API 端点 (`lawmind-local-server.ts`)

- `GET /api/delegations` — 查询委派记录（支持 status/assistantId 过滤）
- `GET /api/delegations/:id` — 委派详情
- `DELETE /api/delegations/:id` — 取消委派
- `GET /api/collaboration-events` — 协作审计事件（支持 since 时间过滤）

### 7. 编排器（Orchestrator）

- `src/lawmind/agent/orchestrator/types.ts`：工作流定义（步骤 + 依赖图 + 审查者）
- `src/lawmind/agent/orchestrator/executor.ts`：工作流执行引擎
  - 拓扑排序依赖图
  - 无依赖步骤并行执行
  - 可选审查步骤（完成后交给审查助手）
  - 结果聚合与报告生成
- `src/lawmind/agent/orchestrator/directive-parser.ts`：自然语言指令解析器
  - LLM 驱动解析（优先）
  - 启发式模式匹配（回退，支持顺序/并行/单步中文指令）
  - `parseAndBuildWorkflow()` 一键从律师指令生成可执行工作流

### 8. 共享记忆 (`src/lawmind/agent/collaboration/shared-memory.ts`)

- `loadCollaborationContext()`：加载协作增强的记忆上下文（基础记忆 + 协作结果 + 事件）
- `saveCollaborationArtifact()`：持久化协作产物到 `workspace/collaboration/<matterId>/`
- `buildCollaborationSummary()`：生成近期协作摘要（可注入 system prompt）
- 审计日志 (`collaboration-audit.jsonl`)：所有跨助手交互全程记录

### 9. 桌面 UI 协作面板

- 侧边栏"助手协作"可折叠区域（badge 显示活跃委派数）
- "委派任务"tab：状态标签 + 来源/目标助手 + 结果预览
- "协作动态"tab：逆序事件时间线

## 设计决策

1. **参照 OpenClaw 而非重新发明**：fire-wait-read、subagent registry、untrusted content wrapping 等核心模式直接对齐 OpenClaw，降低认知负担。
2. **律师控制权**：所有协作由律师授权（`enableCollaboration` 开关），助手不能自发通信。
3. **安全边界**：跨助手结果标记为不可信（`<<<BEGIN_UNTRUSTED_ASSISTANT_RESULT>>>`），防止 prompt injection。
4. **深度限制**：默认最大嵌套 3 层，防止 A→B→C→D 递归失控。
5. **进程内通信**：当前为单进程内 in-memory 路由，未来可扩展为 RPC。

- 编排器支持两种解析路径：LLM（精确）→ 启发式（快速回退）
- 共享记忆产物存储在 `workspace/collaboration/<matterId>/`
